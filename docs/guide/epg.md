# EPG 节目单（使用与配置）

本页说明如何在播放器中使用 EPG（节目单），以及 EPG 与“回看/时移”的关系。

## EPG 是什么

EPG（Electronic Program Guide）是电视节目的时间表。播放器通过 EPG 获取：

- 每个频道当前/已播/未播节目的时间段
- “回看”所需的节目开始/结束时间（用于生成回放 URL）

## EPG 数据来源

播放器支持 XMLTV（含 `.gz`）格式的 EPG 地址。常见来源方式：

- **在 M3U 头部提供** `x-tvg-url`：

```m3u
#EXTM3U x-tvg-url="http://example.com/e.xml"
```

- **在播放器设置中配置 EPG URL**（当 M3U 未提供或需要覆盖时使用）

## 频道如何与 EPG 对上（匹配优先级）

播放器采用**链式多级匹配**策略，按以下顺序尝试匹配：

### 匹配优先级（从高到低）

1. **tvg-id 精确匹配** — 直接用 M3U 的 `tvg-id` 与 EPG XML 的 `channel@id` 对应
2. **tvg-id 清洗后匹配** — 去除 `-`、`_`、空格等特殊字符后再次匹配（如 `CCTV-1` 与 `CCTV1`）
3. **tvg-name 清洗后匹配** — 用 M3U 的 `tvg-name` 与 EPG 的 `display-name` 进行清洗后匹配
4. **channelName 清洗后匹配** — 用 M3U 的频道名称（`display-name`）与 EPG 的 `display-name` 清洗后匹配
5. **智能模糊匹配（兜底）** — 处理 CCTV/卫视等常见命名差异

### M3U 中的相关属性

```m3u
#EXTINF:-1 tvg-id="cctv1" tvg-name="CCTV" group-title="央视" ch-name="CCTV-1 综合",http://example.com/stream
```

| 属性 | 说明 |
|------|------|
| `tvg-id` | 频道唯一标识，用于精确匹配 |
| `tvg-name` | 频道名称标识（独立于显示名称） |
| `ch-name` 或逗号后 | 频道显示名称 |

### 建议

- 尽量在 M3U 中填写正确的 `tvg-id`（最准确的匹配方式）
- 同时填写 `tvg-name` 可以提高匹配成功率
- 频道显示名称保持与 EPG 中的标准名一致或接近

## 播放器内操作（查看节目单）

1. 选择一个频道并播放。
2. 打开 EPG 面板（左侧节目单）。
3. 可切换日期查看不同天的节目。
4. 节目单会根据当前时间显示“正在播放/即将播放/已播出”等状态；右侧使用**芯片式标识**：
   - 直播：红色填充芯片；可点击，等同于点击频道播放直播。
   - 回放：绿色填充芯片；点击节目标题按模板生成回放并播放。
  - 预约：显示“预约”按钮（与芯片同尺寸/圆角），仅未来节目出现；支持“仅提醒/自动播放”。
   - 正在播放：整行高亮；回放播放中左侧有绿色竖线提示。

## EPG 与回看/时移的关系

### 回看

- 回看是“基于节目单”的：点击已播节目后，播放器使用该节目的开始/结束时间去替换 `catchup-source`（或全局模板）中的时间占位符。
- 因此：EPG 越准确，回看越准确。

### 时移

- 时移的时间范围与游标时间会受 EPG 的最早节目时间影响（用于限制可回退的最早边界）。
- 当某时间点能命中节目单时，播放器会优先使用该节目的结束时间作为回放 URL 的结束时间；否则会使用默认时长进行兜底。

## 常见问题

- **有频道但没有节目单**：优先检查 `tvg-id` 是否匹配，或 EPG 地址是否可访问。
- **回看节目点开后时间不对**：多数是 EPG 时区/源端要求的 UTC 与本地时间不一致，建议对照源端要求调整 `catchup-source` 的时间格式（详见 [回看与时移](./catchup-timeshift)）。
