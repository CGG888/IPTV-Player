# EPG 節目單（使用與設定）

本頁說明如何在播放器中使用 EPG（節目單），以及 EPG 與「回看/時移」的關係。

## EPG 是什麼

EPG（Electronic Program Guide）是電視節目時間表。播放器透過 EPG 取得：

- 每個頻道已播/正在播/未播節目的時間段
- 「回看」所需的節目開始/結束時間（用於生成回放 URL）

## EPG 資料來源

播放器支援 XMLTV（含 `.gz`）格式的 EPG 位址。常見提供方式：

- **在 M3U 標頭提供** `x-tvg-url`：

```m3u
#EXTM3U x-tvg-url="http://example.com/e.xml"
```

- **在播放器設定中配置 EPG URL**（當 M3U 未提供或需要覆蓋時使用）

## 頻道如何與 EPG 對上（匹配優先級）

播放器採用**鏈式多級匹配**策略，按以下順序嘗試匹配：

### 匹配優先級（從高到低）

1. **tvg-id 精確匹配** — 直接用 M3U 的 `tvg-id` 與 EPG XML 的 `channel@id` 對應
2. **tvg-id 清理後匹配** — 去除 `-`、`_`、空格等特殊字符後再次匹配（如 `CCTV-1` 與 `CCTV1`）
3. **tvg-name 清理後匹配** — 用 M3U 的 `tvg-name` 與 EPG 的 `display-name` 進行清理後匹配
4. **channelName 清理後匹配** — 用 M3U 的頻道名稱（`display-name`）與 EPG 的 `display-name` 清理後匹配
5. **智慧模糊匹配（兜底）** — 處理 CCTV/衛視等常見命名差異

### M3U 中的相關屬性

```m3u
#EXTINF:-1 tvg-id="cctv1" tvg-name="CCTV" group-title="央視" ch-name="CCTV-1 綜合",http://example.com/stream
```

| 屬性 | 說明 |
|------|------|
| `tvg-id` | 頻道唯一識別碼，用於精確匹配 |
| `tvg-name` | 頻道名稱識別碼（獨立於顯示名稱） |
| `ch-name` 或逗號後 | 頻道顯示名稱 |

### 建議

- 盡量在 M3U 中填寫正確的 `tvg-id`（最準確的匹配方式）
- 同時填寫 `tvg-name` 可以提高匹配成功率
- 頻道顯示名稱保持與 EPG 中的標準名一致或接近

## 播放器內操作（查看節目單）

1. 選擇頻道並播放。
2. 打開 EPG 面板（左側節目單）。
3. 可切換日期查看不同天的節目。
4. 節目單會依當前時間顯示「正在播放／即將播放／已播出」等狀態；右側使用晶片式標識：
   - 直播：紅色填充晶片；可點擊，等同於點擊頻道播放直播
   - 回放：綠色填充晶片；點擊節目標題依模板生成回放 URL 並播放
   - 預約：顯示「預約」按鈕（尺寸與圓角與晶片一致），僅未來節目出現
   - 正在播放：整行高亮；回放播放中左側會顯示綠色豎線提示（深色模式更明顯）

## EPG 與回看/時移的關係

### 回看

- 回看是「基於節目單」的：點選已播節目後，播放器使用該節目的開始/結束時間，替換 `catchup-source`（或全域模板）中的時間佔位符。
- 因此：EPG 越準確，回看越準確。

### 時移

- 時移的時間範圍與游標時間會受 EPG 最早節目時間影響（限制可回退的最早邊界）。
- 若某時間點能命中節目單，播放器會優先使用該節目的結束時間作為回放 URL 的結束時間；否則使用預設時長兜底。

## 常見問題

- **有頻道但沒有節目單**：先檢查 `tvg-id` 是否匹配，以及 EPG 位址是否可存取。
- **回看時間不正確**：多為時區（UTC/本地）或來源端格式要求差異，請對照來源端要求調整 `catchup-source` 的時間格式（詳見 [回看與時移](./catchup-timeshift)）。
