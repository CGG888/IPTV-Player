# EPG 節目單（使用與設定）

本頁說明如何在播放器中使用 EPG（節目單），以及 EPG 與「回看/時移」的關係。

## EPG 是什麼

EPG（Electronic Program Guide）是電視節目時間表。播放器透過 EPG 取得：

- 每個頻道已播/正在播/未播節目的時間段
- 「回看」所需的節目開始/結束時間（用於生成回放 URL）

## EPG 資料來源

播放器支援 XMLTV（含 `.gz`）格式的 EPG 位址。常見提供方式：

- **在 M3U 標頭提供** `x-tvg-url`：

```m3u
#EXTM3U x-tvg-url="http://example.com/e.xml"
```

- **在播放器設定中配置 EPG URL**（當 M3U 未提供或需要覆蓋時使用）

## 頻道如何與 EPG 對上（tvg-id 與匹配）

播放器優先使用 `tvg-id` 匹配；當 `tvg-id` 不一致時，會再嘗試依頻道名稱匹配（含一定的清理/模糊策略）。

建議：

- 盡量在 M3U 中為頻道填寫正確的 `tvg-id`
- 頻道名稱保持與 EPG 標準名一致或接近（避免額外前後綴）

## 播放器內操作（查看節目單）

1. 選擇頻道並播放。
2. 打開 EPG 面板（左側節目單）。
3. 可切換日期查看不同天的節目。
4. 節目單會依當前時間顯示節目狀態。

## EPG 與回看/時移的關係

### 回看

- 回看是「基於節目單」的：點選已播節目後，播放器使用該節目的開始/結束時間，替換 `catchup-source`（或全域模板）中的時間佔位符。
- 因此：EPG 越準確，回看越準確。

### 時移

- 時移的時間範圍與游標時間會受 EPG 最早節目時間影響（限制可回退的最早邊界）。
- 若某時間點能命中節目單，播放器會優先使用該節目的結束時間作為回放 URL 的結束時間；否則使用預設時長兜底。

## 常見問題

- **有頻道但沒有節目單**：先檢查 `tvg-id` 是否匹配，以及 EPG 位址是否可存取。
- **回看時間不正確**：多為時區（UTC/本地）或來源端格式要求差異，請對照來源端要求調整 `catchup-source` 的時間格式（詳見 [回看與時移](./catchup-timeshift)）。
