# Архитектура

Проект разработан на **C# / WPF**, основная архитектура выглядит следующим образом:

- **UI слой**: Основан на WPF (ModernWpf), обеспечивает плавный современный интерфейс.
- **Слой взаимодействия**: Инкапсулирует C API libmpv через `MpvPlayer.cs` с использованием P/Invoke.
- **Слой рендеринга**: Использует `WindowsFormsHost` для размещения дескриптора окна Win32, встраивая вывод mpv в интерфейс WPF, что решает проблемы производительности нативных медиа-элементов WPF.
- **Сервисный слой**:
  - `M3UParser`: Высокопроизводительный парсер на регулярных выражениях, поддерживающий сложные расширенные теги M3U.
  - `EpgService`: Асинхронная загрузка и кэширование EPG в памяти на основе `XmlSerializer`.

## Структура проекта

```text
📂 SrcBox
├── 📂 Models          # Модели данных (Channel, EpgProgram, Source)
├── 📂 Services        # Сервисы (Парсинг M3U, EPG, Управление каналами)
├── 📂 Resources       # Ресурсы (Строки локализации, Стили, Шрифты)
├── 📂 Interop         # Слой взаимодействия с libmpv (P/Invoke)
├── 📄 MainWindow.xaml # Логика главного окна
└── 📄 MpvPlayer.cs    # Обертка ядра плеера
```

## Движок libmpv

Проект зависит от `libmpv-2.dll`.

- **Аппаратное декодирование**: По умолчанию включено `d3d11va`.
- **Проблема со звуком**: Для некоторых источников IPTV с медленным определением аудио установлено `probesize=32` для ускорения старта, что может вызывать кратковременную тишину.
