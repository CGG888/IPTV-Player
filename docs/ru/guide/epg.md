# EPG (использование и настройка)

На этой странице описано, как пользоваться EPG (телегидом) в плеере и как EPG связан с архивом (Catchup) и timeshift.

## Что такое EPG

EPG — это расписание передач. Плеер использует EPG для:

- отображения программы передач (прошлое/текущее/будущее)
- точного времени начала/окончания для генерации URL архива

## Источник EPG

Поддерживается формат XMLTV (в том числе `.gz`). Обычно EPG задают так:

- **В заголовке M3U** через `x-tvg-url`:

```m3u
#EXTM3U x-tvg-url="http://example.com/e.xml"
```

- **В Настройках плеера** (если M3U не содержит EPG или нужно переопределить)

## Сопоставление каналов с EPG (приоритет сопоставления)

Плеер использует **цепочку многоуровневого сопоставления** в следующем порядке:

### Приоритет сопоставления (от высокого к низкому)

1. **Точное совпадение tvg-id** — напрямую сопоставляет M3U `tvg-id` с EPG XML `channel@id`
2. **Сопоставление tvg-id после очистки** — совпадает после удаления специальных символов `-`, `_`, пробелов (например, `CCTV-1` совпадает с `CCTV1`)
3. **Сопоставление tvg-name после очистки** — сопоставляет M3U `tvg-name` с EPG `display-name` после нормализации
4. **Сопоставление channelName после очистки** — сопоставляет M3U имя канала (`display-name`) с EPG `display-name` после нормализации
5. **Умное нечёткое сопоставление (запасной вариант)** — обрабатывает распространённые вариации именования CCTV/TV Station

### Атрибуты M3U

```m3u
#EXTINF:-1 tvg-id="cctv1" tvg-name="CCTV" group-title="CCTV" ch-name="CCTV-1",http://example.com/stream
```

| Атрибут | Описание |
|---------|----------|
| `tvg-id` | Уникальный идентификатор канала для точного сопоставления |
| `tvg-name` | Идентификатор имени канала (независимый от отображаемого имени) |
| `ch-name` или после запятой | Отображаемое имя канала |

### Рекомендации

- Заполняйте корректный `tvg-id` в M3U (наиболее точный метод сопоставления)
- Также заполняйте `tvg-name` для повышения вероятности успешного сопоставления
- Держите отображаемые имена каналов близкими к каноническим именам в EPG или соответствующими им

## Действия в плеере (просмотр EPG)

1. Включите канал.
2. Откройте панель EPG (список программ).
3. Переключайте дату для просмотра других дней.
4. Список показывает статус программ в зависимости от текущего времени; справа используются чип‑бейджи:
   - Прямой эфир: красный заполненный чип; кликабелен, мгновенно запускает текущий канал в прямом эфире
   - Архив (Replay): зеленый заполненный чип; клик по названию передачи генерирует URL архива по шаблону и запускает воспроизведение
   - Напоминание: кнопка «Напоминание» (того же размера/скругления, что и чипы); появляется только для будущих передач
   - Сейчас воспроизводится: вся строка подсвечивается; при воспроизведении архива слева показывается зеленая полоска для лучшей заметности

## Связь с архивом / timeshift

### Архив

- Архив “привязан” к EPG: при выборе прошедшей передачи плеер берет ее время начала/окончания и подставляет в `catchup-source` (или глобальный шаблон).

### Timeshift

- Диапазон timeshift может ограничиваться временем самой ранней передачи из EPG.
- Если время курсора попадает в передачу, плеер предпочитает использовать конец этой передачи как время окончания; иначе используется запасной вариант по длительности.

## Частые проблемы

- **Канал работает, но EPG пустой**: проверьте `tvg-id` и доступность URL EPG.
- **Время архива неверное**: чаще всего проблема в часовом поясе (UTC vs локальное) или требуемом формате. См. [Архив и Timeshift](./catchup-timeshift).
