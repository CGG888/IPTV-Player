# Настройки HTTP/RTSP Header

На этой странице объясняется, как настроить пользовательские HTTP Headers и параметры RTSP в проигрывателе для поддержки специальных источников потокового вещания.

## Обзор функции

Проигрыватель поддерживает пользовательские параметры Header для потоков HTTP/HTTPS и протокола RTSP:

- Пользовательский User-Agent для источников, требующих определённый идентификатор браузера
- Проверка Referer или Cookie для защищённых источников
- Аутентификация RTSP и настройка режима передачи

## Настройки HTTP Headers (для потоков HTTP/HTTPS)

### Поддерживаемые поля

| Поле | Описание | Пример |
|------|----------|---------|
| User-Agent | Идентификатор браузера | `Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36...` |
| Referer | Страница источника | `https://example.com/player` |
| Cookie | Cookie сессии | `session=abc123; token=xyz` |

### Формат настройки

В интерфейсе настроек каждая строка — один Header в формате `Поле: Значение`:

```
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36
Referer: https://example.com/
Cookie: session=abc123
```

### Сценарии использования

- **Авторизация IPTV**: Некоторые операторы или платформы требуют определённый User-Agent
- **Защищённые источники**: Потоки, требующие проверку Referer
- **Поддержание сессии**: Сохранение сессии с помощью Cookie

## Настройки RTSP

### Режимы передачи

| Режим | Описание | Сценарий использования |
|-------|----------|------------------------|
| TCP | Режим по умолчанию, высокая проходимость, хорошая стабильность | Большинство сетевых окружений |
| UDP | Режим низкой задержки, требует поддержки UDP в сети | Локальная сеть, высококачественные внутренние источники |
| HTTP Tunnel | Режим HTTP-туннеля для сред с HTTP-прокси | Источники за корпоративным файрволом |

### User-Agent

Специфичный для RTSP User-Agent, некоторые устройства/платформы используют другие идентификаторы:

```
VLC/3.0.18 Libmpv
```

### Аутентификация

Поддерживается стандартная аутентификация RTSP:

- **Имя пользователя**: Имя пользователя для аутентификации RTSP
- **Пароль**: Пароль для аутентификации RTSP (хранится в зашифрованном виде)

## Функция тестирования

### Как использовать

1. Введите полный URL воспроизведения (включая данные аутентификации) в поле "Тестовый URL"
2. Нажмите кнопку "Тест HTTP" или "Тест RTSP"
3. Проигрыватель начнёт тестирование с текущими настроенными параметрами Header
4. Проверьте журнал отладки проигрывателя для получения подробной информации

### Примечания

- Тестовый URL должен содержать полный префикс протокола (`http://` или `rtsp://`)
- Для RTSP URL, требующих аутентификацию, можно встроить учётные данные прямо в URL: `rtsp://user:password@host:port/path`
- Тестирование не изменяет сохранённые настройки, а только проверяет правильность конфигурации

## Расположение настроек

1. Откройте "Настройки" → вкладка "Воспроизведение"
2. Прокрутите вниз до раздела "Настройки HTTP/RTSP Header"
3. Настройте HTTP Headers и параметры RTSP по необходимости
4. Нажмите "Сохранить" для применения настроек

## Часто задаваемые вопросы

### Настроенные Headers не работают

- Проверьте, начинается ли URL с `http://` или `https://` (HTTP Headers применяются только к потокам HTTP/HTTPS)
- Убедитесь, что формат Header правильный: по одному в строке в формате `Поле: Значение`
- Проверьте журнал отладки, чтобы подтвердить правильность отправки Headers

### Ошибка подключения RTSP

- Попробуйте переключить режим передачи (TCP/UDP/HTTP Tunnel)
- Убедитесь, что имя пользователя и пароль указаны верно, или попробуйте встроить аутентификацию прямо в URL
- Проверьте, не блокирует ли файрвол порт RTSP (по умолчанию 554)

### Безопасность пароля

- Пароли RTSP зашифрованы с помощью Windows DPAPI и хранятся локально
- Пароли расшифровываются только при необходимости, открытый текст никогда не сохраняется в памяти
